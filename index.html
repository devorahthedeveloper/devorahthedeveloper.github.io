Proto (a front end framework)

Reinvented FXCM's approach to front-end development.

  • Object-oriented JavaScript
  • Responsive Design
  • HTML5/CSS3
  • Pub/Sub
  • Information Architecture

Proto is a comprehensive front-end framework and accompanying JavaScript abstraction layer I designed and architected for use on FXCM's retail sites.

In designing Proto for my team, my goal was to build a system of reusable components that:

  1. Encouraged consistent code structure, design and site user experience.
  2. Reduced the time it took to code new web content.
  3. Empowered all developers, especially junior developers, to build complex functionality.
  4. Was flexible enough to scale over time and accomodate growing business / technical needs.
View framework docs See framework on fxcm.com
Division of labor when building Proto

I developed Proto's underlying philosophy. This included determining the roster of components (modules) and how they would interact with each other. I also architected the macro grids (a reduced set of generic containers), the page-level grids, file and folder naming conventions, content templates and the methodology for constructing CSS classes. Here, the work of Nicole Sullivan's OOCSS, Jonathan Snook's SMACSS, and the BEM methodology were illuminating.

I then divided the coding between myself and the other developers on my team. I chose this method to reinforce the modularity of the framework. We were each responsible for building modules that could stand on their own but shared common interface patterns with the other modules.

We also built a companion JavaScript abstraction layer for applying common behavior such as modals, video players, ajax forms and accordions, all via CSS classes. And lastly, we built a documentation site to catalogue the system.

Results of adopting Proto

Within a few weeks of lauching Proto on fxcm.com, we saw metrics improve across the board:

  • Bounce rate dropped 30-40% on the homepage
  • Increase in "site stickiness" by 1-2 pages per user
  • 21% lift in lead conversions
  • Nearly double account conversions
  • Time to code standard pages dropped by 70%.

Replacing special characters

Lightweight JavaScript that identifies and replaces special characters within form submission.

  • JavaScript
  • Data manipulation

This script enables you to replace special characters within user-collected form data. It can be called within a form submit handler or upon any other event of your choice.

View demo View code on Github
Notes:

I coded this script to correct an issue with FXCM lead-generation ajax forms. The webservice handling these ajax forms could not process accented characters, and consequently, we were losing valuable leads in countries where accented characters (such as: ç,ü­,æ,ß, etc) are common.

We didn’t have the option to revise the webservice in the short term, so I built this script to replace problematic characters with predefined safe replacements upon submit. For example, "Über­grö­ße" "becomes "Ueber­groe­sse".

With the script in place, we saw a lift in conversions.

View the open-sourced demo on github. The original script is also still in place on FXCM forms.
